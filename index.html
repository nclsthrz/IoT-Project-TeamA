Overview

Reflex Challenge Game.
The Reflex Challenge is a timing-based game where the player must press a button as close as possible to a target moment. The OLED screen displays the elapsed time and the current attempt number (Try X/10).

Hardware components:

  • ESP32 microcontroller
  • OLED display
  • 4 colored LEDs
  • 4 push buttons
  • Buzzer

Method

I implemented the Reflex Challenge game using system timers to accurately measure reaction time. The deviation from the target time is calculated in milliseconds and displayed on the OLED screen.

I implemented the scoreboard without using EEPROM because persistent storage caused reliability issues. Scores are therefore managed per session to ensure stable gameplay.

The project was initially developed on Wokwi. Merging the three games did not work correctly in Visual Studio Code at first. After restructuring and debugging the code, I successfully merged all three games and ran the full project locally in VS Code.
